Gym Membership Management

Track members, fees, attendance, and trainer assignments

Overview

This system manages gym members, monitors their fee payments, tracks attendance, and assigns trainers. Here's the workflow:

  1. Register new members with personal details and membership plan.
  2. Record and track payment status for each member.
  3. Log daily attendance for performance and usage tracking.
  4. Assign trainers to members for guided workouts.

Data is stored in members.json and attendance.json.

Detailed Specification

Objective

Track gym members, fees, attendance, and trainer assignments.

Core Classes

  • Member – name, age, membership type, contact details.
  • Trainer – name, specialization (e.g., cardio, weights).
  • Gym – manages members, trainers, and attendance.

Core Features

  1. Register members with chosen membership type.
  2. Assign trainers to members.
  3. Record daily attendance of members.

Example Flow

  1. John registers for a 3-month membership.
  2. Trainer “Alice – Yoga Specialist” is assigned to him.
  3. Attendance marked daily.
  4. After 10 days of absence → John appears in “Inactive Members” list.

1. Member Registration

When a new member joins, record the following:

  • Member ID – Unique identifier (e.g., GM101).
  • Name & Contact Info – Address, phone number, email.
  • Membership Type – Monthly, quarterly, yearly.
  • Start Date & Expiry Date – For membership validity.

Saved in members.json for future reference.

2. Fee Tracking

Track each member’s fee status:

  • Paid – Member has cleared payment for the current period.
  • Pending – Payment due soon.
  • Overdue – Payment date has passed.

Helps maintain financial stability and ensure timely payments.

3. Attendance Recording

Mark attendance daily:

  1. Member scans ID or is marked manually by staff.
  2. Log includes date and time of entry.
  3. Data is saved in attendance.json.

4. Trainer-Member Assignment

Assign specific trainers to members for personalized plans:

  • Each trainer can manage multiple members.
  • Assignments can be updated based on schedules or goals.

JSON File Structures

members.json

[ { "member_id": "GM101", "name": "Rahul Sharma", "contact": "rahul@example.com", "phone": "+91-9876543210", "membership_type": "Yearly", "start_date": "2025-01-01", "expiry_date": "2025-12-31", "fee_status": "Paid" } ]

[ { "member_id": "GM101", "date": "2025-08-12", "time": "07:45 AM" }, { "member_id": "GM101", "date": "2025-08-13", "time": "07:50 AM" } ]
