Flight Booking Simulation

A complete walkthrough of how this system works

Overview

This simulation mimics a mini airline booking system. Here's how it works in real-world terms:

  1. The airline adds flight details to the system (route, time, seats).
  2. Customers book tickets for those flights, and the system tracks seats.
  3. Customers can cancel bookings which frees up their seats.
  4. The system can generate a list of passengers for any flight.

All this data is stored in two files: flights.json and passengers.json.

Detailed Specification

Objective

Manage flights, passengers, and bookings.

Core Classes

  • Flight – number, origin, destination, seat capacity, available seats.
  • Booking – passenger name, seat number, flight reference.
  • FlightSystem – manages flights and bookings.

Core Features

  • Add flights with schedules and capacity.
  • Book seats for passengers.
  • Cancel bookings (seat becomes available).

Example Flow

  1. Flight AI-202 → Delhi → Mumbai → 60 seats.
  2. Passenger A books a ticket → seat #15 reserved.
  3. Passenger cancels → seat #15 is freed again.

1. Flight Schedule Management

Admins add new flights with the following info:

  • Flight number – Unique ID like AI101
  • Source and Destination – e.g., Delhi → Mumbai
  • Date and Time – When the flight departs
  • Total Seats – e.g., 150 seats on the plane

The system saves this data to flights.json. No passengers are added yet.

2. Seat Booking

Steps when a passenger books a seat:

  1. Passenger selects a flight by flight number.
  2. The system checks if available seats > 0.
  3. If yes, it asks for passenger name and assigns a seat (e.g., 12A).
  4. Updates flights.json to decrease available_seats.
  5. Saves passenger info in passengers.json.

This ensures only available seats are booked.

3. Booking Cancellation

If a passenger cancels their booking:

  1. The system finds their entry in passengers.json.
  2. Removes their record.
  3. Increases the available_seats in flights.json.

This keeps flight data accurate.

4. Passenger List Reports

To view who is flying on a flight:

  1. Select a flight number (e.g., AI101).
  2. The system filters passengers.json by that flight number.
  3. Displays all passengers and their seat numbers.

This is useful for generating flight manifests or check-in lists.

JSON File Structures

flights.json

[ { "flight_number": "AI101", "source": "Delhi", "destination": "Mumbai", "datetime": "2025-08-14 10:00", "total_seats": 150, "available_seats": 20 } ]

passengers.json

[ { "name": "Shivam Bansal", "flight_number": "AI101", "seat_number": "12A" }, { "name": "Mohini Bansal", "flight_number": "AI101", "seat_number": "14B" } ]
