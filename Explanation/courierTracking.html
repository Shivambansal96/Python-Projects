Courier & Parcel Tracking

Track parcels from sender to receiver with real-time status updates

Overview

This application helps a courier company record parcels, track their status, and confirm deliveries. Here's how it works:

  1. Staff registers new parcels with sender and receiver details.
  2. Status is updated as the parcel moves through transit stages.
  3. Delivery is confirmed when the receiver signs for the parcel.
  4. Users can search parcels by tracking ID to check status.

Data is stored in parcels.json and customers.json.

Detailed Specification

Objective

Register parcels and track their delivery status.

Core Classes

  • Parcel – ID, sender, receiver, weight, delivery status.
  • CourierService – manages parcels (register, update status, search).

Core Features

  • Register parcels with unique ID.
  • Update parcel status (In Transit, Out for Delivery, Delivered).
  • Search parcels by ID.

Example Flow

  1. Parcel ID 101 created → Sender: A, Receiver: B.
  2. Status updated: In Transit → Out for Delivery → Delivered.
  3. Receiver confirms delivery → status closed.

1. Parcel Registration

When a new parcel is sent, the following details are recorded:

  • Parcel ID – Unique tracking number (e.g., CP12345).
  • Sender Details – Name, address, contact info.
  • Receiver Details – Name, address, contact info.
  • Weight & Dimensions – For pricing and logistics.
  • Current Status – Initially set to Registered.

Stored in parcels.json, linked to sender/receiver in customers.json.

2. Status Update

As the parcel moves, the status is updated to reflect its journey:

  • In Transit – Parcel is on the way.
  • Out for Delivery – Parcel is with delivery staff.
  • Delivered – Parcel has reached the receiver.
  • Delayed – Unexpected delay reported.

These updates are made in parcels.json so tracking is always accurate.

3. Delivery Confirmation

When the receiver gets the parcel:

  1. Delivery staff marks the parcel as Delivered.
  2. Optionally captures the receiver's signature or photo.
  3. The delivery date and time are recorded.

4. Parcel Search by ID

Anyone can track a parcel by entering its tracking ID:

  1. The system looks up parcels.json for a match.
  2. Displays current status, sender, and receiver info.
  3. Shows full journey history if available.

JSON File Structures

parcels.json

[ { "parcel_id": "CP12345", "sender_id": 1, "receiver_id": 2, "weight": "2kg", "dimensions": "30x20x10cm", "status": "In Transit", "history": [ {"date": "2025-08-10", "status": "Registered"}, {"date": "2025-08-11", "status": "In Transit"} ] } ]

customers.json
